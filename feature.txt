Required Inputs:

Algorithm & Logic Instructions (The 3 Phases):

Phase 1: The Comparatist (Multi-Genome & Orthology Engine)

Action: Ingest all genomes and annotations. Run an ortholog clustering algorithm (e.g., using a lightweight wrapper for MMseqs2 or parsing pre-computed BLAST/Diamond all-vs-all results).

Logic: Construct a gene presence/absence matrix using pandas.

Filtration: Automatically partition the genes into the Core Genome (present in >95% of strains) and the Accessory/Unique Genome (present in <10% of strains or strain-specific).

Output: Pass only the sequence coordinates of the Accessory/Unique genes to Phase 2.

Phase 2: The HGT Detective (Horizontal Gene Transfer Engine)

Action: Analyze the unique genes from Phase 1 to determine if they are of foreign origin (Alien/HGT).

Logic: Use scikit-learn (e.g., Isolation Forest or PCA) to detect genomic anomalies based on:

Tetranucleotide (k-mer) frequency deviations compared to the genome's average.

GC content skew.

Proximity to Mobile Genetic Elements (MGEs) like transposases, integrases, or phage regions in the GFF file.

Filtration: Flag genes/regions that pass the anomaly threshold as Alien_HGT_Regions.

Phase 3: The AI Discoverer (Secondary Metabolite Deep Learning Engine)

Action: This is the payoff. Instead of running heavy computations on the entire genome, deploy a Deep Learning model (e.g., a PyTorch-based Protein Language Model / ProtBERT or a Graph Neural Network) only on the translated protein sequences within the Alien_HGT_Regions.

Logic: (For this initial build, create a PyTorch skeleton/mock-inference module that scores the probability of a sequence being a NRPS/PKS or novel BGC).

Result: Identify high-confidence novel BGCs acquired via recent evolutionary events.
